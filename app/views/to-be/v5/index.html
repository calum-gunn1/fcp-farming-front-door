Version 5

In this round of testing, we focused on SFD (Single front door) and RPS (Rural payments service) running simultaneously, viewing and updating details and updates/statuses of schemes/grants.

We based this round of testing on what is in scope in the 'Capybara' phase of delivery.

We wrote the following How might we statements to guide our design thinking:

  • How might we launch a new version of RPS in parallel to the existing service?
  • How might we help users navigate the service?
  • How might we convey confirmations and updates so users don't need to chase for details?
  • How might we show all details about an account so that information can be found and updated as necessary?
